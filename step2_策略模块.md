基于现有仓库与共用约束，请开发“策略模块”。

目标：
构建一个可扩展的策略与信号生成框架，严格与数据层、回测层、风控层、执行层解耦。

设计原则：
- 策略层负责“看数据 -> 产出信号 / 目标仓位”
- 策略层不负责成交仿真
- 策略层不负责风控裁剪
- 策略层不直接操作券商接口

必须实现的能力：

一、策略抽象层
请设计并实现以下抽象或等价结构：
- BaseStrategy
- StrategyContext
- StrategyRunner
- SignalGenerator 或 TargetGenerator
- StrategyConfig

策略输入：
- 来自 data 模块的历史 bars / as-of 数据窗口
- 当前时点信息
- 可选的组合状态或上下文

策略输出：
- Signal 或 TargetPosition
- 输出格式必须统一到 core 中约定的数据结构

二、策略执行范式
请支持以下能力：
- warmup window
- rebalance frequency（例如 daily / weekly）
- 参数配置化
- 缺失数据处理
- 多标的策略兼容
- 明确防止未来函数

三、至少实现两个样例策略
请至少提供这两个可运行样例：
1. 双均线策略
   - 短均线 > 长均线 产生多头信号
   - 短均线 < 长均线 产生平仓或空信号（按你设计，但要统一）
2. 横截面动量排名策略
   - 对标的池按过去 N 期收益排序
   - 选前 K 个生成目标仓位或强度信号

要求：
- 两个样例都能直接跑在数据模块提供的样例数据上
- 参数通过配置传入
- 样例要尽量简洁可读，作为以后开发模板

四、对外接口
至少实现一个高层入口，让外部可以这样使用：
- 加载数据
- 初始化策略
- 运行策略
- 拿到 signals / target positions

五、测试
请至少覆盖：
- 双均线信号正确性
- warmup 行为
- 缺失数据处理
- 动量排名输出结果
- 不同 rebalance frequency 的行为
- 严格不使用未来数据

六、示例与文档
请提供 examples：
1. 运行双均线策略
2. 运行动量排名策略
3. 输出 signal / target 示例结果

文档说明至少包括：
- 如何新增一个策略
- 策略类需要实现哪些方法
- 数据层与策略层如何衔接
- 为什么当前实现不会产生未来函数

实现约束：
- 策略层不允许耦合成交逻辑
- 策略层不允许直接写资金和持仓账本
- 尽量保持接口简洁
- 允许后续接入更多因子和机器学习模型
- 如果需要微调 core 接口，可做最小必要调整并保持兼容

交付验收标准：
- examples 可直接运行并输出 signals 或 target positions
- pytest 可以通过
- 至少两个策略样例可用
- 下游 backtest 模块可以直接消费策略输出
- 文档足够支持后续扩展新策略

完成后请输出：
1. 改动文件清单
2. 策略框架设计说明
3. 两个样例策略的运行方式
4. 测试覆盖了哪些关键点
5. 后续如何继续扩展更多策略
